Tutorial 11 — Inter-Individual Variability and Virtual Populations¶

Prerequisites: Tutorials 01–10

A single simulation represents one hypothetical patient. Real phage therapy trials enrol tens to hundreds of patients who differ in bacterial burden, immune function, phage pharmacokinetics, and many other parameters. Inter-individual variability (IIV) modelling generates a virtual patient population (VPOP) that captures this heterogeneity, producing realistic distributions of outcomes.

What you will learn

  1. IIV distributions: Fixed, LogNormal, Normal, Uniform
  2. IIVSpec — mapping distributions to model parameters
  3. InitialConditions and the "ic." path prefix for variable starting values
  4. VirtualPopulation — generating 100–1000 patient-specific configs
  5. TrialRunner — parallel batch simulation
  6. Outcome metrics: time_to_clearance, nadir_count, bacterial_auc, final_resistant_fraction
  7. plot_trajectories, plot_metric_distributions, plot_time_to_event

2 IIV distributions¶

pbisim provides four distribution types for varying parameters across patients:

Distribution Constructor Use when
Fixed(v) constant — no variability Reference parameter
LogNormal(cv=...) right-skewed, always positive PK rates, bacterial load
Normal(mean, sd) symmetric, can be negative Additive errors
Uniform(lo, hi) uniform Bounded parameters

Coefficient of variation (CV) for LogNormal: CV ≈ σ in log-space. A CV of 0.3 means ±30% variability is typical. PK parameters commonly have CV = 30–50%; bacterial inoculum CV = 50–100% in clinical settings.

3 Defining IIV with IIVSpec¶

IIVSpec maps model parameter paths to (distribution, transform) pairs.

Parameter paths mirror ModelConfig field names. The special "ic." prefix maps to InitialConditions fields (initial bacterial count, phage count, etc.).

Transform types:

  • "multiplicative": sampled value multiplies the base value (use for rates)
  • "additive": sampled value is added to the base value (rare)

Clinical variability modelled here:

  • growth_rates: patient-to-patient variation in bacterial growth (colonised sites with different nutrient supplies, strain sub-population variability)
  • ic.B: pre-treatment bacterial burden varies by 3–4 orders of magnitude
  • adsorption_rates: phage–bacteria binding varies with receptor expression level
In [3]:
iiv = IIVSpec({
    "growth_rates":     (LogNormal(cv=0.3), "multiplicative"),   # ±30% CV
    "adsorption_rates": (LogNormal(cv=0.4), "multiplicative"),   # ±40% CV
    "ic.B":             (LogNormal(cv=0.8), "multiplicative"),   # ±80% CV inoculum
})
print("IIV paths defined:")
for path, (dist, mode) in iiv.specs.items():
    print(f"  {path}: {type(dist).__name__}  ({mode})")

4 Base config + InitialConditions¶

InitialConditions carries the central (population-mean) initial values. IIVSpec then samples patient-specific deviations from these.

In [4]:
# Base config: single-strain phage therapy model
base_cfg = (
    ModelBuilder(n_bacteria=1, n_phages=1, n_latent=5)
    .with_growth_rates(1.0, bacteria_to_resource_ratio=1e9)
    .with_phage_params(adsorption_rates=2e-9, burst_sizes=50.0,
                       latent_periods=0.5, phage_decay_rates=0.1)
    .with_nutrient(monod_constant=0.3, recycle_fraction=0.0)
    .with_dose_schedule(DoseSchedule([
        DoseEvent(time=0, amount=5e8, target="phage", index=0),
    ]))
    .with_initial_conditions(
        InitialConditions(B=np.array([3e7]), P=np.zeros(1), S=1.0)
    )
    .build()
)

# Virtual population: 200 patients sampled from iiv

sample_cfgs = vpop.generate_cohort(5)
print(f"Sample growth rates (first 5): "
      f"{[round(c.growth_rates[0],3) for c in sample_cfgs]}")

5 TrialRunner — batch simulation¶

TrialRunner applies a model factory function to each patient's ModelConfig, runs all simulations in parallel, and computes outcome metrics.

Model factory: a function factory(cfg) → PBIModel. It must read the initial conditions from cfg.initial_conditions so that patient-level variability in ic.B and ic.P propagates into the simulation.

Outcome metrics (all can be customised):

  • time_to_clearance(threshold): first time B drops below threshold (h)
  • nadir_count: minimum bacterial density reached (cells/mL)
  • bacterial_auc: area under the bacterial density curve (cells·h/mL)
  • final_resistant_fraction: fraction of bacteria that are resistant at t_end
In [5]:
def make_model(cfg):
    ic = cfg.initial_conditions
    return PBIModel(
        cfg,
        initial_B = ic.B if ic else np.array([3e7]),
        initial_P = np.zeros(1),   # phage added via dose_schedule
        initial_S = float(ic.S) if ic else 1.0,
    )

runner = TrialRunner(
    vpop       = vpop,
    model_factory = make_model,
    t_end      = 72.0,
    dt         = 0.5,
    metric_fns = default_metrics(),
)
trial = runner.run()

print(f"Patients simulated: {trial.n_succeeded + trial.n_failed}")
print(f"Failures (ODE did not converge): {trial.n_failed}")
print("\nMetric summary:")
print(trial.metrics.describe().round(2))

6 Population-level visualisations¶

Three complementary visualisations provide a complete picture of the virtual trial outcome:

  1. plot_trajectories: per-patient time courses (spaghetti plot) with a quantile ribbon showing the median and interquartile range.

  2. plot_metric_distributions: histograms of each scalar outcome metric across all 200 virtual patients.

  3. plot_time_to_event: empirical CDF of time-to-clearance — the percentage of patients who achieved bacterial clearance by each time point. This is the clinical equivalent of a Kaplan–Meier curve.

Biological interpretation of the spread:

  • Wide trajectory ribbons indicate that the treatment outcome is highly sensitive to patient-specific parameters — a signal that personalised dosing or patient stratification may be needed.
  • A bimodal metric distribution often reveals an underlying bistability (e.g., the immune threshold demo from Tutorial 04).

7 Summary¶

Object Role
IIVSpec({path: (dist, transform), ...}) Map parameter paths to variability distributions
InitialConditions(B=..., P=..., S=...) Central initial values for the virtual population
VirtualPopulation(cfg, iiv, n_patients, seed) Sample n patient-specific configs
TrialRunner(vpop, model_fn, t_end, dt, metric_fns) Run all patients in parallel
trial.metrics DataFrame (n_patients × n_metrics)
plot_trajectories(trial) Spaghetti + quantile ribbon
plot_metric_distributions(trial) Metric histograms
plot_time_to_event(trial, metric) Empirical CDF

Key insight: IIV modelling allows estimation of how likely a given treatment is to succeed across a realistic patient population — a more clinically relevant question than the outcome for a single nominal patient.

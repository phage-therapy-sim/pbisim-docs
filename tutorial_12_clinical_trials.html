Tutorial 12 — Multi-Arm Clinical Trials¶

Prerequisites: Tutorial 11

With TrialRunner, each virtual patient population (VPOP) ran a single treatment. The matched-twins design in ClinicalTrial runs multiple treatment arms over the same patients — guaranteeing that differences in outcomes reflect the treatment, not between-patient variability.

What you will learn

  1. TreatmentArm — dose schedules and initial conditions per arm
  2. ClinicalTrial — matched-twins execution
  3. ClinicalTrialResult — accessing per-arm results and comparing arms
  4. Visualising and interpreting the arm comparison
  5. How matched twins reduce statistical noise

2 Base configuration and IIV¶

We use the same single-strain model as Tutorial 11. The three treatment arms will differ only in their dosing schedules.

base_cfg = (
    ModelBuilder(n_bacteria=1, n_phages=1, n_latent=5, n_depth=2)
    .with_growth_rates(1.0, bacteria_to_resource_ratio=1e9)
    .with_phage_params(adsorption_rates=2e-9, burst_sizes=50.0,
                       latent_periods=0.5, phage_decay_rates=0.1,
                       adsorption_rates_dormant=2e-11)
    .with_dormancy(dormancy_rate=0.1, resuscitation_rate=0.05,
                   dormancy_signal="nutrient", resuscitation_signal="nutrient")
    .with_antibiotic("cipro", k_elim=0.5, Vc=15.0, k12=0.4, k21=0.1,
                     emax=4.0, ec50=0.5, hill=1.0,
                     emax_dormant=0.8, ec50_dormant=2.0)
    .with_nutrient(monod_constant=0.3, recycle_fraction=0.0)
    .with_initial_conditions(
        InitialConditions(B=np.array([5e7]), P=np.zeros(1), S=1.0)
    )
    .build()
)

iiv = IIVSpec({
    "growth_rates":     (LogNormal(cv=0.3), "multiplicative"),
    "adsorption_rates": (LogNormal(cv=0.4), "multiplicative"),
    "ic.B":             (LogNormal(cv=0.7), "multiplicative"),
})

def make_model(cfg):
    ic = cfg.initial_conditions
    return PBIModel(cfg,
                    initial_B=ic.B if ic else np.array([5e7]),
                    initial_P=np.zeros(1),
                    initial_S=float(ic.S) if ic else 1.0)

3 Defining treatment arms¶

Each TreatmentArm specifies only what differs between arms. Typically this is the dose_schedule; the base_cfg parameters (growth rate, PK, etc.) are shared across all arms via the matched-twins mechanism.

Trial design:

  • Control: no treatment — establishes natural disease course
  • Phage monotherapy: single IV-equivalent phage bolus at t = 0
  • Combination: phage + ciprofloxacin (q12h, 3 days)

Matched-twins rationale: by assigning the SAME patient to all three arms, the comparison isolates the treatment effect from patient heterogeneity. This is equivalent to a crossover design and dramatically reduces the number of virtual patients needed to achieve statistical power.

arm_control = TreatmentArm(
    name="Control (no treatment)",
    dose_schedule=None,
)

arm_phage = TreatmentArm(
    name="Phage monotherapy",
    dose_schedule=DoseSchedule([
        DoseEvent(time=0, amount=5e8, target="phage", index=0),
    ]),
)

arm_combo = TreatmentArm(
    name="Phage + ciprofloxacin",
    dose_schedule=DoseSchedule([
        DoseEvent(time=0,  amount=5e8,  target="phage",      index=0),

4 Running the ClinicalTrial¶

ClinicalTrial generates ONE VPOP and runs each arm over the same patients. This is the matched-twins design.

In [4]:
trial = ClinicalTrial(
    base_config   = base_cfg,
    iiv_spec      = iiv,
    n_patients    = 150,
    arms          = [arm_control, arm_phage, arm_combo],
    model_factory = make_model,
    t_end         = 72.0,
    dt            = 0.5,
    seed          = 42,
)
result = trial.run()

print("Arms:", result.arm_names)
for arm_name in result.arm_names:
    arm_result = result[arm_name]
    n_cleared = arm_result.metrics["time_to_clearance"].notna().sum()
    nadir_med = arm_result.metrics["nadir_count"].median()
    print(f"  {arm_name:35s}  "
          f"clearance: {n_cleared:3d}/150 ({100*n_cleared/150:.0f}%)  "
          f"nadir median: {nadir_med:.1e}")

5 Visualising the arm comparison¶

result.compare() returns a merged DataFrame with an 'arm' column, convenient for plotting and statistical analysis.

The three visualisations below show:

  1. Trajectory fan: how bacterial density evolves in each arm
  2. Nadir distributions: how deeply each treatment suppresses bacteria
  3. Time-to-event CDF: how quickly each treatment achieves clearance

Expected biological interpretation:

  • Control arm: bacteria grow to capacity, no decline
  • Phage monotherapy: bacteria suppressed; regrowth possible if all phage are consumed before complete clearance
  • Combination: phage handles bulk replicating population; antibiotic targets dormant bacteria that phage cannot reach → synergistic, deeper, more durable suppression

6 Quantifying the treatment benefit¶

result.compare() provides a single DataFrame across all arms — useful for computing effect sizes, p-values (with sufficient n), or input to statistical packages.

A simple effect-size comparison: the log10 fold-reduction in bacterial nadir relative to the control arm.

In [6]:
merged = result.compare()
print("Merged DataFrame columns:", list(merged.columns))
print(f"Total rows (n_patients × n_arms = 150 × 3): {len(merged)}")

# Log10 fold-reduction in nadir vs control
ctrl_nadir_med = result["Control (no treatment)"].metrics["nadir_count"].median()

print("\nTreatment effect — median bacterial nadir vs control:")
for arm_name in result.arm_names:
    arm_nadir = result[arm_name].metrics["nadir_count"].median()
    fold = ctrl_nadir_med / max(arm_nadir, 1)
    print(f"  {arm_name:35s}  median nadir = {arm_nadir:.1e}  "
          f"  ({np.log10(fold):.1f} log10-fold reduction vs control)")

7 Summary¶

Object Role
TreatmentArm(name, dose_schedule) One arm's dosing specification
ClinicalTrial(base_config, iiv_spec, n_patients, arms, ...) Matched-twins trial
result["arm_name"] TrialResult for one arm
result.compare() Merged DataFrame with 'arm' column
result.arm_patient_configs dict[arm, list[ModelConfig]] per arm

The matched-twins advantage: with n = 150 matched pairs, the standard error of the nadir difference is √2 times smaller than for an independent groups design of the same size — equivalent to doubling the sample size for free.

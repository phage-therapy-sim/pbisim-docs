Tutorial 04 — Immune Response¶

Prerequisites: Tutorials 01–03

This tutorial focuses exclusively on bacteria–immune interactions — no phage — so that the effect of each immune module is unambiguous. Phage therapy combined with immunity is covered in later tutorials once each component is well understood in isolation.

What you will learn

  1. The Imm state variable and the bac_total saturation denominator
  2. 'innate' module — stimulation/decay ODE with Imm-proportional killing
  3. 'hill' module — static saturation killing, no Imm ODE
  4. 'custom' module — fully user-controlled callable
  5. Bistable immune clearance — how initial bacterial density determines whether the immune system succeeds or fails (threshold ≈ 10⁷ CFU/mL)
  6. Immune killing of dormant bacteria (imm_kill_rate_D)

T_END = 36.0   # h — enough to see clearance or persistence

3 'innate' module¶

The innate module models a dynamic immune effector Imm:

dImm/dt = Σ imm_stim_rate[i]·B[i] / (imm_stim50 + B_tot)  −  imm_decay_rate·Imm
kill_B[i] = imm_kill_rate[i] · Imm · B[i] / (imm_kill50 + B_tot)

B_tot = Σ B[i] + Σ I — shared saturation denominator for both stimulation and killing. At very high bacterial loads, the Imm response saturates and per-bacterium killing falls.

Key parameters:

Parameter Role
imm_stim_rate How strongly each cell activates the immune response
imm_kill_rate Immune kill rate coefficient
imm_stim50 / imm_kill50 Half-saturation bacterial densities
imm_decay_rate Immune effector half-life

fig, axes = plt.subplots(1, 2, figsize=(11, 4))

# Correct parameter magnitude:
#   kill_B = imm_kill_rate * Imm * B / (imm_kill50 + B_tot)
#   Per-capita kill rate = imm_kill_rate * Imm_ss / (imm_kill50 + B)

#   Values 4+ orders of magnitude below this are invisible against bacterial growth.
for (kr, color, label) in [
    (0,     "C0", "No immunity"),
    (5e5,   "C1", "Weak   (kill_rate = 5e5)"),
    (3e6,   "C2", "Medium (kill_rate = 3e6  ≈ μ threshold)"),
    (2e7,   "C3", "Strong (kill_rate = 2e7)"),
]:
    cfg = (
        BASE.with_immunity(
            immune_module  = "innate",
            imm_stim_rate  = 1.0,
            imm_kill_rate  = kr,
            imm_decay_rate = 0.15,
            imm_stim50     = 5e6,
            imm_kill50     = 5e6,
        ).build()
    )
    res = solve_ode(

4 'hill' module¶

The Hill module uses a static saturation formula — Imm plays no role (it is frozen at its initial value):

kill_B[i] = imm_max · B[i] / (imm_kill50 + B_tot)

The denominator uses population-level B_tot, so immune saturation reflects total bacterial load — not per-strain density.

This is mathematically simpler and useful when immune dynamics are not the focus. It is also the right starting point for the bistable analysis below.

5 Bistable immune clearance threshold¶

A hallmark of innate immunity is bistability: the same immune parameters can either clear bacteria entirely or fail, depending on the initial bacterial load. This arises from competition between bacterial growth and immune killing.

Mechanistic analysis (Hill module, logistic growth):

dB/dt = μ · (1 − B/K) · B  −  imm_max · B / (imm_kill50 + B)

Per-capita net growth f(B) = μ·(1−B/K) − imm_max/(imm_kill50 + B) has:

  • f(0) = μ − imm_max/imm_kill50 — negative when imm_max > μ · imm_kill50 (immune kills faster than bacteria grow at very low density)
  • f rises as B increases (killing saturates), crosses zero at an unstable threshold B*
  • Returns negative near K (logistic suppression dominates)

This creates two stable states: B = 0 (immune clearance) and B ≈ K (bacterial persistence), separated by the unstable threshold B*.

Parameters chosen so that B* ≈ 10⁷ CFU/mL:

Parameter Value Role
imm_max 2×10⁷ imm_max / imm_kill50 = 2 > μ = 1 → clearance at low B
imm_kill50 10⁷ Sets the threshold density
μ 1.0 h⁻¹ Growth rate
K 10⁹ Logistic ceiling

6 'custom' module¶

Provide a callable with the signature:

def my_immune(S, B, I, Imm, config):
    dImmdt  = ...          # float — right-hand side of the Imm ODE
    kill_B  = ...          # ndarray[n] — per-cell killing for active bacteria
    kill_I  = ...          # ndarray[L, n, m] — per-cell killing for infected bacteria
    return (dImmdt, kill_B, kill_I)

The example below implements a simple adaptive threshold response: killing is negligible until Imm exceeds a threshold, then turns on sharply.

In [7]:
def adaptive_immune(S, B, I, Imm, config):
    B_tot   = float(np.sum(B)) + 1e-30
    # Imm grows proportional to bacterial load, decays otherwise
    dImmdt  = 3.0 * B_tot / (5e6 + B_tot) - 0.3 * Imm
    # Killing activates once Imm crosses 0.3
    k_eff   = 5e-7 * max(0.0, Imm - 0.3)
    kill_B  = k_eff * B
    kill_I  = k_eff * I
    return (dImmdt, kill_B, kill_I)

cfg_custom = (
    BASE.with_immunity(immune_module="custom",
                       immune_function=adaptive_immune)
    .build()
)

fig, axes = plt.subplots(1, 2, figsize=(11, 4))
for B0, color in [(1e5,"C0"),(1e7,"C1"),(1e8,"C2")]:
    res = solve_ode(PBIModel(cfg_custom, initial_B=np.array([B0]),
                             initial_P=np.zeros(0), initial_Imm=0.0),
                    t_end=T_END, dt=0.25)
    lbl = f"B₀ = {B0:.0e}"
    axes[0].semilogy(res.time, np.maximum(res.get("B0"), 1), color=color, label=lbl)
    axes[1].plot(res.time, res.get("Imm"), color=color, label=lbl)

axes[0].set(xlabel="Time (h)", ylabel="Cells/mL", title="Active bacteria")
axes[1].axhline(0.3, ls=":", color="k", lw=0.8, label="Kill threshold (Imm=0.3)")
axes[1].set(xlabel="Time (h)", ylabel="Imm", title="Immune effector")
for ax in axes: ax.legend(fontsize=9)
plt.suptitle("Custom adaptive immune response — threshold-activated killing")

7 Immune killing of dormant bacteria (imm_kill_rate_D)¶

There is experimental evidence that phagocytes can kill dormant bacteria, though at a lower rate than active cells. Setting imm_kill_rate_D extends immune killing to the D (dormant) and H (hibernating) compartments.

A single rate is used for both D and H because dormancy suppresses phage synthesis in H, removing the mechanistic basis for distinguishing the two.

When imm_kill_rate_D is set, D and H are also added to bac_total (the immune saturation denominator), reflecting full immune recognition.

In [8]:
BASE_DORM = (
    ModelBuilder(n_bacteria=1, n_phages=0, n_depth=3)

8 Summary¶

Module Imm ODE Killing formula Use when
'innate' Yes imm_kill_rate · Imm · B / (imm_kill50 + B_tot) Full immune dynamics
'hill' No (frozen) imm_max · B / (imm_kill50 + B_tot) Simple saturation, bistability analysis
'custom' User-defined User-defined Threshold, memory, non-standard models

The bistable clearance threshold (Section 5) is a critical model property: imm_max / imm_kill50 > μ is the condition for immune clearance at low bacterial density. The threshold B* separates the basins of attraction of the two stable states (clearance and persistence).

imm_kill_rate_D: extends killing to dormant bacteria; D and H then enter bac_total, coupling dormant load to immune saturation.
